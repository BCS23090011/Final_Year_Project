Limitations & Future Work

An honest assessment of what the data can and cannot tell us —
and how the analytical framework scales to real-world applications.

Important distinction: The analytical methods employed in this study — including Random Forest with SHAP explainability, K-Means clustering, and Self-Organising Maps — demonstrated valid and interpretable results within the scope of the survey dataset. The primary constraints lie not in the methodology but in data scale. Every limitation below traces back to a single root cause: a survey of n=170 players is inherently constrained relative to the scale of the modern gaming market.

Known Limitations

Four Acknowledged Limitations

⚠️ Limitation 1 — Churn Model Constrained to Demographic Features Only

The combined churn model was constrained to five shared demographic features (age, play years, gender, friend influence) because behavioural variables such as login frequency, spending, and event participation were only collected from active players. This limited the model's predictive depth, explaining the moderate AUC scores (LR=0.708, RF=0.648) and the counterintuitive result where LR outperformed RF — a pattern consistent with RF's known underperformance on low-dimensional feature spaces.

⚠️ Limitation 2 — Survey Design Oversight: Missing Retrospective Behavioural Data

A limitation of the current survey design is the absence of retrospective behavioural questions for churned players (e.g., login frequency, spending, and event participation prior to disengagement). Without shared behavioural features across both groups, the combined churn model was constrained to demographic variables only. Future studies should include retrospective behavioural items in the churned player questionnaire to enable feature-aligned modelling across active and churned populations.

⚠️ Limitation 3 — K=2 Clustering Lacks Granularity

Due to the limited sample size (n=126 active players) and the highly imbalanced retention target (91/126 intending to continue, only 11 at-risk), K=2 was selected to ensure minimum cluster viability. The resulting binary segmentation aligns with foundational high/low engagement distinctions in retention literature. Finer segmentation (K=3–5) requires a larger, more balanced dataset and is recommended for future work.

⚠️ Limitation 4 — SOM Quality Above Ideal Thresholds

The SOM produced a Quantization Error of 1.6229 and Topographic Error of 0.4206, both exceeding ideal thresholds (QE < 1.0, TE < 0.2). This is attributable to the small sample size (n=126) relative to the 8×8 grid, which limits the density of training signals per neuron. The SOM results should therefore be interpreted as an exploratory topological visualisation rather than a definitive cluster structure.

Root Cause Analysis

Survey Data vs Server-Side Log Data

The most fundamental constraint is data source — not analytical method. Here is what changes when server-side log data replaces survey data.

Dimension This Study (Survey) Ideal Future (Server Log)
Sample size n=126 / n=44 Millions of records per day
Behavioural data Player self-report (subjective) Objective ground truth
Time dimension Single-point snapshot Continuous time series
Class balance Severely imbalanced (at-risk: 11/126) Natural distribution, controllable sampling

Future Directions

How This Framework Scales

🚀 Priority 1 — Server-Side Log Data Integration

The most significant avenue for future improvement lies in replacing survey-based self-reported data with server-side behavioural logs. Log data would provide objective, high-frequency records of login patterns, session duration, in-game spending, and event participation — eliminating self-report bias and enabling temporal feature engineering (e.g., 30-day login trend, consecutive days played). With sufficient log data, the class imbalance problem would be substantially reduced, enabling reliable identification of at-risk players and real-time churn intervention pipelines.

🚀 Priority 2 — Retrospective Survey Items for Churned Players

Adding retrospective behavioural questions to the churned player survey — specifically asking about login frequency, spending habits, and event participation before disengagement — would enable feature-aligned modelling across both groups. This would dramatically improve the combined churn model and unlock cross-group SHAP comparisons.

🚀 Priority 3 — Larger Sample for Finer Segmentation

With n ≥ 500 active players, K-Means segmentation at K=3–5 would be viable, enabling more granular player persona development. A balanced sample with sufficient at-risk players (ideally n ≥ 50 per class) would also resolve the Class 0 identification failure that limited this study.

🚀 Priority 4 — Real-Time Churn Intervention Pipeline

The ideal production system: Server log data feeds a continuously retrained RF+SHAP model. Each player receives a daily churn probability score. When a player's score crosses a threshold, a personalised re-engagement trigger fires (push notification, in-game reward, limited event access) — targeting the specific features SHAP identifies as that player's primary churn risk drivers.
